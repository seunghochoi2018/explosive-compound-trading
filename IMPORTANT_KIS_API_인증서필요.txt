## 에러: APBK1269 (주문시간 입력오류)

실제 원인: 공인인증서 미등록!

## 해결 방법:

1. KIS Developers 포털 접속
   https://apiportal.koreainvestment.com

2. 서비스 신청 화면으로 이동

3. 인증서 등록 메뉴 찾기

4. 공동인증서(공인인증서) 등록

5. 등록 완료 후 API 주문 가능

## 출처:
매수매도방법.txt (96-98줄)

## 확인 사항:
- 잔고 조회: 인증서 불필요 (조회만 가능)
- 주문 (매수/매도): 인증서 필수!
- 사용자가 직접 구매한 SOXL 17주: 한투 앱에서 수동 구매
- API 자동매매: 인증서 등록 후 가능

## 2025-10-17 추가 발견: SOXL 주문 성공 조합!

### ✅ 실제 작동하는 파라미터:
- **OVRS_EXCG_CD: "AMEX"** (NASD 아님!)
- **PDNO: "SOXL"** (티커 그대로, A980679 사용 금지!)
- **OVRS_ORD_UNPR: 소수점 2자리** (예: "40.94")
- **ORD_DVSN: "00"** (지정가)
- **Hashkey 필수**

### 테스트 결과:
- 매도 성공: 주문번호 0031895400 @ $40.94
- 매수 성공: 주문번호 0031896216 @ $40.78

### 주의사항:
1. 잔고 조회 시 ovrs_excg_cd가 "AMEX"로 표시됨
2. 가격은 반드시 소수점 2자리 (f"{price:.2f}")
3. Hashkey 없으면 실패
4. PDNO에 A980679 사용 시 APBK0656 에러 발생
